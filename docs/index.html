CLI · Ghost · WeChat

ghostwriter-cli Markdown → Ghost → WeChat publishing pipeline

写一次 Markdown，发布到 Ghost 博客，再一键同步到微信公众号草稿箱。

✎ Markdown
👻 Ghost
💬 WeChat
00

安装

$ pip install ghostwriter-cli

或从源码安装：git clone https://github.com/yinguobing/ghostwriter

01

核心能力

Markdown → Lexical

将 Markdown 转换为 Ghost 原生 Lexical 格式。发布后在编辑器里可自由编辑，支持标题、代码块、表格、列表。

图片自动上传

文内图片自动上传至微信永久素材库，替换为 CDN 地址，无需手动处理。

三层白名单

标签、属性、样式三层过滤，确保生成的 HTML 安全合规，通过微信公众号的内容审核。

不兼容标签转换

智能处理微信不支持的 HTML：table → div、ol/ul → 前缀段落、链接 → text[url] 格式。

02

快速上手

配置

通过环境变量或命令设置 Ghost 和微信公众号的 API 密钥：

terminal 
ghostwriter config ghost.api_url https://yinguobing.com/blog
ghostwriter config ghost.admin_key_id xxxxxxxx
ghostwriter config ghost.admin_key hex_secret
ghostwriter config wechat.appid your_appid
ghostwriter config wechat.secret your_secret

发布文章

terminal 
# 列出 Ghost 文章
ghostwriter list

# 发布 Markdown 到 Ghost 草稿
ghostwriter publish article.md --draft

# 发布后自动同步到微信草稿箱
ghostwriter publish article.md --wechat

# 同步已发布文章到微信
ghostwriter sync <article-id>

# 预览微信 HTML（不创建草稿）
ghostwriter sync --preview <article-id>
03

处理管道

Ghost → 微信的 HTML 转换经过多层处理：

  1. 1
    图片上传到微信永久素材
    遍历文中所有图片，上传到微信服务器，替换为 CDN 地址。
  2. 2
    三层白名单过滤
    标签、属性、样式逐层清理，只保留微信支持的子集。
  3. 3
    代码块保护与恢复
    清洗前用占位符保护代码块，清洗后恢复，保留语言标记。
  4. 4
    不兼容标签转换
    table → div、ol/ul → 前缀段落、hr → 分隔线、链接 → text[url]。
  5. 5
    默认样式补全
    段落间距、图片边距、代码块样式等默认值，确保微信阅读体验。
